Naïve Bayes Lab

In this lab you train a Naïve Bayes classifier — the same type of algorithm used in spam filters. You decide what the model learns, and can then see how the mathematics behind the algorithm works when it classifies new emails.

① Train

Click ⚡ Generate to automatically populate the model with example emails. You control how many and the balance between spam and legitimate email.

② Model

See what the model has learned — which words lean toward spam and which toward legitimate email, and how strongly.

③ Classify

Type in an email and press Enter. You get a result showing the model's confidence, and you can follow all the calculations behind the classification.

Create training data

Generates example emails from a built-in bank. Each time you click, more examples are added to the training set, so you can build it up gradually. Vary the balance to see how uneven training data affects the model.

Tokenization and filtering

Before the model can compute probabilities, the text must be split into individual words — this is called tokenization. Words that are too common to say anything about the class (stop words) and very short words are then filtered out. Only the remaining words are used in classification.

Used in classification Stop word — filtered out (too common) Too short — filtered out (≤2 characters)

Note: no lemmatization is performed — "winning" and "winner" are counted as different words. Real systems often reduce words to their base form to improve accuracy.
